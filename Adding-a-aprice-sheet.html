Adding a Price Sheet


 

To access Price Sheet functions, navigate using the steps outlined below.

From the SA Module Main Menu, select the Price Sheet Functions link

Select the Maintain Price Sheets menu option from the Price Sheet Functions menu. This will take you to a browse view with a list of existing price sheets.

Select any of the existing price sheets by pressing ‘enter’ to get to a detail screen.

Select the “Add” option from the display screen bounce bar.

From here select the type of price sheet you would like to add. The choices are Standard or Promotion. The differences between each will be explained in detail below.

On the blank price sheet detail screen:

  • For a Standard price sheet, use the drop down menu to select an existing customer.
    • If the customer has multiple addresses, select the address to use for shipping from the Ship To drop down menu.
    • Select the Freight On Board option. These options will be discussed in more detail below.
  • For a Promotional price sheet, select one of the pre-existing price sheets which have already had the Allow Promo toggle set.

Enter the price sheet title. By default, the title will be the company name of the customer, but this value can be edited if necessary.

  • If you are adding a Standard price sheet, you will be prompted on whether or not to allow this price sheet to serve as a promo via the Allow Promo toggle. The default value is 'on'.

Choose your ship range - the date range for which this price sheet is being created. The default effective (starting) date is the current date; the default expiration date is the effective date + 364 days.

Enter the Full Load Weight. The default is 42,000 lbs. This value can be edited as necessary.

Enter the desired order quantity via the radio check box - either full pallets, full layers or as desired.

Enter the minimum shipment unit load volume for the price sheet, and add a note.

If FOB is delivery to customer dock, then the note will default to "APPLIES ONLY TO P/U ORDERS". This message can be edited as needed.

Enter the prepaid minimum unit load value and add a note.

Enter the Lead Time in business days, and add a note if needed.

Add a P/U Allowance note.

  • If FOB is delivery to customer dock, message defaults to "See pickup allowance price sheet". The message can be edited.
  • If FOB is customer pick up, message defaults to 'N/A - THIS PRICING IS "CUSTOMER PICKUP"'. The message can be edited.

Enter any price protection details. The default message is "0 DAY ADVANCE NOTICE OF PRICE INCREASE. NO PRICE PROTECTION ON PRICE DECLINE." Edit as needed.

If you are entering a standard price sheet, enter any special instructions in the memo field. The default message is: "ULV = UNIT LOAD VALUE - ULV OF 1 = 1 FULL PALLET. CUSTOMER MUST ORDER EACH ITEM IN FULL PALLET QUANTITIES. CLAIMS FOR PROMOTIONAL DISCREPANCIES MUST BE SUBMITTED WITHIN 12 MONTHS FROM THE EXPIRATION DATE OF THE PROMOTION. CLAIMS FOR PRICING DISCREPANCIES MUST BE SUBMITTED WITHIN 12 MONTHS OF THE INVOICE DATE. ANY CLAIMS SUBMITTED BEYOND THE 12 MONTH PERIOD WILL BE DISALLOWED." Edit as needed.

Enter the heading for the level 1. The default value is "All".

  • If you are only entering one pricing level, you also have the option of editing the upper range boundary, provided that it is a value greater than 0. The default value is 999.

(If required)Enter the heading for levels 2 - 6.

  • If you choose to enter more than one level, you will need to set the lower range boundaries for each level, and the upper range boundary for the last level. Setting the lower range boundary on a level will automatically set the upper range boundary for the preceding level to 1 less than the entered value. The lower range of successive levels must be higher than the low range of the preceding level.

If you do not wish to allow invoicing on the price sheet, set No Invoicing toggle to "On". Default position is off.

If you require broker data to be entered prior to allowing orders, set Broker Data toggle to "On". Default position is off.

Select the number of significant digits to use in filtering f/g records. Default value is 10. NOTE: Value must be either 10 or 12 digits.
