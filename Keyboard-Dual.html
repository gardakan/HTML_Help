Keyboard Navigation In A Dual List Screen


 

Dual List Screens are screens with two column-separated lists. One list, typically occupying the upper-left side of the screen is the so-called parent list. The second list, to the right of the parent, is the child list.
 

 
When opening to a Dual List Screen, the cursor focus starts on the parent list. As the user moves the cursor between records on the parent list, the contents of the child list will display records associated with the currently highlighted parent record. NOTE: This document assumes the user is familiar with the basic List Screen keyboard navigation commands, described here. Most navigation is identical in both single and dual list pages. This page describes the additional features unique to a Dual List Screen.

Navigating a Dual List Screen
KEY
 FUNCTION
PARENT LIST CHILD LIST
CTRL + TAB Switches cursor focus to child list. Switches cursor focus to parent list.

When first opening a dual list page, the bounce bar at the bottom of the screen is associated with the parent list, which can be activated as usual by pressing the TAB key.  Functions such as "Add", "Change", or "Delete" are tied to parent records.  There is usually a menu option on the bounce bar with the same (or very similar) name as the title of the child list - selecting this option, or by using the keyboard shortcut CTRL + TAB will move cursor focus from the parent list to the child list.  In addition, a new bounce bar will be displayed at the bottom of the screen whose functions are tied to the child list.  Pressing the TAB key will alternate focus between the child list and the child bounce bar.  To return to the parent list, select the bounce bar menu item with the same name as the title of the parent list, or use the keyboard shortcut CTRL + TAB.
Parent list bounce bar with the "Subclass" menu item highlighted. The title of the child list is SubClasses. Selecting this will move focus to the child list.
Child list bounce bar with the "Class" menu item highlighted. The title of the parent list is Classes. Selecting this item will return focus to the parent list.

File associations - Adding a record

Adding a record in the child list will automatically associate it with the currently highlighted parent record.

Adding a record to the parent list will only create that entry - if any child list associations need to be made, they should be added to the child list after creating the parent list record while that record is highlighted in the parent list view. After adding a new parent record, the child list view will still be displaying records attached to the previously highlighted parent record. Refresh the child list view by pressing the down or up arrow keys to navigate to an adjacent record and then back again:
Highlighted parent record (left) and associated child record (right)
Newly added parent record now highlighted (left), but child list view (right) still showing association with previously highlighted parent record.
Using up arrow, highlight focus moves to a different record on the parent list (left), and the child list (right) is refreshed to show this record's associations.
Moving the cursor back to the newly added parent record (left) with the down arrow, the child list view is refreshed to display a blank column.

File associations - Deleting a record

Deleting a child record removes the entry and its association with a parent record.
NOTE: Deleting a parent entry deletes both the parent and any associated child records from the child list.
