Customer Maintenance


 

This document assumes familiarity with basic Detail Screen navigation.  Please refer to the Detail Screen page for more information about keyboard commands and common concepts.

The Customer Maintenance screen shows detailed information about Jasper Product's customers.  Below is a description of some of the data fields including their place in the context of the Jasper Products sales cycle.

Field Field Type Description Use In Jazz
Remit To Drop-Down Menu Remit To refers to the Jasper Products return address (address to send payment/correspondence/shipments etc). Remit to address is needed for printing invoices, sales orders and credit memos.
Send Invoice Drop-Down Menu This field describes the preferred method of invoicing the customer. The options are Fax, Phone, Web, Email, US Mail, and Overnight. The Send Invoice field is only used in the customer detail screen.
Ship Via Drop-Down Menu Used to determine appropriate shipping method for orders. This information is used in many parts of Jazz, including EDI (Electronic Data Interchange) functions (data taken from EDI file), sales order consolidation, truck booking, prepay and bill (company prepays), sales order creation, printing invoices/memos/sales orders, and updating invoices. Ship Via data is used in Sales Order/Order Planning/Invoice/credit Memo maintenance.
FOB Drop-Down Menu Freight on Board – determines way in which orders are shipped/picked up, whether customer or JP is responsible for shipping. The two main options are DEL, or Delivery, and OD, or Our Dock. DEL clients have finished goods delivered to them, and the responsibility to arrange and cover shipping costs falls to Jasper Products. OD clients are responsible for the scheduling and cost of arranging shipping to pick the finished goods from the Jasper Products dock. FOB data is used in many shipping and order processing functions throughout Jazz, such as Truck Booking, order invoicing, pallet ID's/Bills of Lading and Credit Memos.
Type Drop-Down Menu This field describes the nature of the type of business carried out by the customer. The customer type is only used in the customer detail screen, though the list of available customer types can be edited via the Customer Types menu option in the Customer Functions submenu.
Price List Drop-Down Menu A Price List is a customer-specific list of available products and pricing. They can be tailored to suit individual client needs. Please see section on adding a Price Sheet for additional details. Price List data displayed in the drop-down menu is a list of available price sheets by price sheet code number.
 
Price Sheets can be added, edited or deleted by following these instructions. The codes are also needed for attaching items to a price list, and for running a Price Sheet Overview report.
Upcharge % Numeric text field. This is a % consolidation fee/freight upcharge percentage. Upcharge % data is used in invoicing functions, including Prepay and Bill and Voucher posting.
Net Due Days Numeric text field. The number of days in which a customer has to pay in full. This data is used for various invoicing functions, including calculating whether a customer is on credit hold, keeping track of overdue invoices, and displaying the order terms on the customer display screen.
Paymnt Days Numeric text field. The number of days alotted for a discounted payment. This data is used to help calculate discount payments, which is used in printing invoices, credit memos, vouchers and cash receipts, as well as for displaying the order terms on the customer detail screen.
Paymnt Disc Numeric text field. Payment discount percentage for paying within the time frame specified in the Paymnt Days field. This data is used to calculate discount amounts, and is used when creating orders, printing invoices, receipts and credit memos, as well as in displaying order terms on the customer detail screen.
Sales Tax Numeric text field. The percentage sales tax to add to customer orders. The sales tax is used when creating customer orders.
Fin Chg Numeric text field. Percentage charged on past due amounts. This data is only used in the customer detail screen.
Terms Drop-Down Menu / Alphanumeric text field Depending on the values entered into the Net Due Days, Paymnt Days, and Paymnt Disc fields, the Terms field will display differently.
 
If no value was entered for the number of Net Due Days or the amount entered was 0, and the Paymnt Disc field is 0, a popup box will prompt you to enter the terms from a drop down menu:
 

 
If no value was entered for the number of Net Due Days or the amount entered was 0, and the Paymnt Disc field is greater than 0, a popup box will prompt you to enter the percentage terms from a drop down menu:
 

 
If the number of Paymnt Days is greater than 0 and the Paymnt Disc field is greater than 0, the field will display in the format of
 
"<payment discount percentage>% <no. of days allotted for discounted payment>, <no. of net due days> Days":
 

 
If the number of Paymnt Days is equal to 0 and the Paymnt Disc field is greater than 0, the field will display in the format of
 
"<payment discount percentage>% Net, <no. of net due days> Days":
 

 
NOTE: The Terms field can be overwritten with a custom note if applicable
 
 The terms are displayed when creating an order, and is displayed on invoices, receipts and reports.
Credit Limit Numeric text field. The credit limit is the maximum unpaid balance allowed for the customer. The credit limit is used in calculating customer financials, which includes displaying available credit when performing sales order line item pricing or checking whether a customer is on credit hold when creating an order, and displaying credit info when making collection calls.
Limit Date Date field This field is the date at which the credit limit expires. Credit limit expiration date is checked when pricing sales order line items and when creating a customer order.
Full pallets only Checkbox Check this box if only full pallet orders can be shipped to the customer This data is only used in the customer detail screen.
Ship on Hold Checkbox Check this box if finished goods on Q/A hold can be shipped to the customer. The Ship on Hold data is used for maintaining system Ship-Tos. This checkbox, along with a similar value from the Ship To address information and a valid email address are required to permit a ship-on-hold.
 
These values are checked when scanning a Pallet ID and setting up a BOL.
Email BOL Checkbox Check this box if the Bill Of Lading is to be emailed to the customer. This data is included in the customer contact info, and is checked by the system for automatic sending of an advance shipping notice/BOL.
Inactive Checkbox Check this box if the customer is no longer an active Jasper Products customer. This value is used to filter out inactive customers from searches, and to prevent orders from being placed to customers who are no longer active.
Pricing Mode Radio Button Pricing Mode is the framework from which the pricing is decided. The choices are Private Label, Production Date, and Purchase Order. This value is used to determine whether pricing needs to be changed or renewed when placing an order.
Customer class Radio Button This field describes the customer classification into market segments, which is divided into two main choices (Co-Pack or Trade) and a third Other option if the customer does not meet the other two class types. Please see this page on Customer Classes for additional details. This field is used throughout Jazz where market segment information is needed. This includes:
  • Finished goods, including mismatched inventory, barcode inventory reporting, and repacking/restacking,
  • Attaching items to a Price Sheet
  • Order invoicing
Customer group Drop-Down Menu Customers can be grouped together if several use the same formulas / place identical orders. Usually group members are part of the same large company, but it can be any group. Groups are created under the Customer Group menu option in the SA module. This data is referenced throughout Jazz to find specific associations belonging to a specified customer group, such as a common Contract, R/M coordinator, CSR or Aged R/M Extension Method.
Grace Days After Q/A Numeric text field. Number of grace days allowed after finished goods have been released from QA hold. Grace days are used for calculating f/g Storage Fees.
Double Stack Pallet Rate/Pallet Numeric text field. Double stack weekly rate per pallet – double stacking describes when one pallet is stacked on top of another Double Stack Pallet Rate is used to calculate f/g Storage Fees.
Single Stack Pallet Rate/Pallet Numeric text field. Single stack weekly rate per pallet – single stacking describes storage of a solo pallet. Single Stack Pallet Rate is used to calculate f/g Storage Fees.
Lot Bar Code Prefix Alphanumeric text field 2 character finished goods lot bar code prefix. If entered, a lot number bar code is printed on the finished goods pallet label. The Lot Barcode Prefix is used in the printing of Pallet ID barcodes.
UPC-A Prefix Numeric text field. Single-digit (0-9) prefix for UPC-A barcode. UPC-A is the retail barcode, which identifies the manufacturer and specific product. Rest of barcode is managed by Uniform Code Council. The UPC-A Prefix is used in the printing of Pallet ID barcodes.
DC Label Checkbox Check this box if the customer requires a distribution center label on all pallets (almost exclusively used on Walgreens/Soylent orders). This DC Label flag is checked in several places throughout Jazz, including:
  • a specific search when loading pallets onto a Walgreens/Soylent sales order (checking that the correct Walgreens shipping label has been printed)
  • Printing a Walgreens-specific BOL
  • Displaying Ship-To information (4-digit Walgreens warehouse ID)
Load ID Checkbox Check this box if the customer requires a distribution center label and a warehouse load ID on that label. The Load ID is used to in setting up/printing Walgreens/Soylent Pallet ID labels, in scheduling traffic appointments (determining if a customer requires a warehouse load ID), and truck booking consolidation.
